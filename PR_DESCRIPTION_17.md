# Pull Request: Практическое занятие №17 - A11y-аудит и обеспечение доступности

## Описание изменений

Реализованы все требования практического занятия №17 по A11y-аудиту и обеспечению доступности для страницы контактов.

## Выполненные задачи

### ✅ 1. Семантическая разметка
- Использованы правильные HTML5 теги (`<form>`, `<fieldset>`, `<legend>`)
- Все интерактивные элементы семантически корректны
- Правильная иерархия заголовков (`<h1>`-`<h4>`)
- Логическая структура документа с секциями
- Добавлен skip-link для быстрой навигации

### ✅ 2. Доступность форм
- Все поля имеют связанные `<label>` с `for`/`id`
- Обязательные поля помечены `aria-required="true"` и `required`
- Поля с ошибками имеют `aria-invalid="true"` и `aria-describedby`
- Группы полей объединены в `<fieldset>` с `<legend>`
- Есть визуальные и текстовые индикаторы обязательных полей
- Добавлены подсказки через `aria-describedby`

### ✅ 3. Навигация с клавиатуры
- Все интерактивные элементы доступны с Tab
- Логический порядок фокуса
- Видимый и четкий индикатор фокуса (`:focus-visible`)
- Клавиша Enter/Space работают на всех элементах
- Escape очищает форму (с подтверждением)
- Ловушка фокуса внутри формы (циклическая навигация Tab)

### ✅ 4. ARIA-атрибуты
- Правильное использование `aria-label`, `aria-labelledby`, `aria-describedby`
- Состояния элементов (`aria-required`, `aria-invalid`, `aria-current`)
- Живые регионы (`aria-live="polite"`) для динамического контента
- Логические роли (`role="alert"`, `role="status"`, `role="contentinfo"`)
- `aria-hidden="true"` для декоративных иконок

### ✅ 5. Цвет и контраст
- Соотношение контрастности минимум 4.5:1 для текста
- Цвет не используется как единственный способ передачи информации
- Состояния элементов различимы без цвета (границы, подчеркивания)
- Поддержка высокого контраста через `@media (prefers-contrast: high)`

### ✅ 6. Дополнительные улучшения
- Валидация в реальном времени
- Живые регионы для объявлений скринридерам
- Поддержка `prefers-reduced-motion` для пользователей с ограничениями
- Автозаполнение через `autocomplete`
- Правильные `rel` атрибуты для внешних ссылок
- Минимальный размер кликабельных элементов (44x44px)

## Изменённые файлы

- `pages/contacts.html` - полностью переработана с учетом всех требований доступности
- `styles/a11y.css` - новый файл стилей для улучшения доступности
- `scripts/a11y-form.js` - новый JavaScript файл для улучшения доступности формы
- `A11Y_AUDIT_REPORT.md` - отчет по аудиту доступности

## Технологии

- HTML5 (семантические теги, ARIA)
- CSS3 (focus-visible, prefers-contrast, prefers-reduced-motion)
- JavaScript (валидация, управление фокусом, живые регионы)
- WCAG 2.1 Level AA compliance

## Особенности реализации

- **Валидация в реальном времени**: Поля валидируются при потере фокуса и вводе
- **Живые регионы**: Динамические сообщения объявляются скринридерам
- **Ловушка фокуса**: Фокус циклически перемещается внутри формы
- **Улучшенные сообщения об ошибках**: Конкретные сообщения для каждого типа ошибки
- **Поддержка различных предпочтений**: Высокий контраст, уменьшенная анимация
